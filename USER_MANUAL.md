# Пользовательское руководство Ney Firmware

Добро пожаловать в детальное руководство по установке и использованию **Ney Firmware** — кастомной прошивки безопасности для Raspberry Pi, которая превращает ваше устройство в защищенный VPN-маршрутизатор (Double-VPN).

---

## 1. Установка Raspberry Pi OS на SD-карту

Прежде чем устанавливать прошивку Ney, нужно подготовить чистую операционную систему на вашей Raspberry Pi.

### Шаг 1: Скачивание Raspberry Pi Imager
Скачайте и установите официальную утилиту [Raspberry Pi Imager](https://www.raspberrypi.com/software/) для Windows, macOS или Ubuntu.

### Шаг 2: Настройка конфигурации (OS Customization)
1. Вставьте microSD-карту в кардридер компьютера.
2. Откройте Raspberry Pi Imager.
3. В разделе **CHOOSE DEVICE**, выберите вашу модель Raspberry Pi.
4. В разделе **CHOOSE OS** выберите **Raspberry Pi OS (64-bit)**. Рекомендуется версия "Lite" (без графики), если вы планируете работать только через терминал/SSH.
5. Нажмите кнопку с шестеренкой ⚙️ (или `Ctrl+Shift+X`), чтобы открыть настройки предварительной конфигурации:
   - Установите галочку **Установить hostname** (например, `Ney-Router`).
   - Установите галочку **Включить SSH** (выберите аутентификацию по паролю).
   - Укажите **имя пользователя и пароль** (обязательно запомните их, они понадобятся для входа по SSH).
   - *Опционально:* Настройте Wi-Fi, чтобы Pi сразу подключилась к вашему домашнему роутеру, пока прошивка не настроит собственный `wlan1`.

### Шаг 3: Запись образа
Нажмите **CHOOSE STORAGE**, выберите вашу флешку, и затем **WRITE**. Дождитесь успешного окончания записи.

---

## 2. Установка Ney Firmware

Вставьте флешку, подключите к Raspberry Pi питание и Ethernet-кабель (если не настроили Wi-Fi заранее).

1. Узнайте IP-адрес вашей Raspberry Pi в панели вашего домашнего роутера.
2. Подключитесь к Pi по SSH (используйте Терминал в Mac/Linux или PuTTY в Windows):
   ```bash
   ssh ваш_юзер@IP_МАЛИНКИ
   ```
3. Войдите под правами root или используйте `sudo` для запуска установки:
   ```bash
   git clone https://github.com/confarik/ney-firmware.git /tmp/ney
   cd /tmp/ney
   sudo bash install.sh
   ```
4. Дождитесь окончания (5–10 минут). Прошивка сама скачает зависимости, установит DKMS драйвера и обновит мосты (`br0`, `br1`). Конец установки сопровождается успешным сообщением.
5. Для первой очистки и раздачи сети запустите: `sudo resetney`.

## 3. Главное Меню и Информационные Панели

В прошивке встроено множество утилит с единой стилистикой и алиасами (их можно вызывать прямо из консоли одной буквой).

- `iney` — **Главное Интерактивное Меню**. Отсюда можно управлять всеми VPN, прокси и настройками в удобном графическом FZF-интерфейсе без запоминания других команд.
- `infoney` — **Инфо-панель**. Показывает красивый Dashboard: ваш текущий публичный IP (и через что он идет), состояние работы интерфейсов (`tun0`, `br0`), температуру и потребление RAM.
- `stopney` — **Экстренная остановка** (Panic Button). Мгновенно убивает все процессы `openvpn`, `wireguard`, `ssney`, `ckney` и т.д., обрывая любую связь с удаленными узлами.

---

> [!IMPORTANT]
> **ПРАВИЛО ДОБАВЛЕНИЯ ПУТЕЙ (SFTP/SCP)**:
> Чтобы загрузить свои конфиги VPN/Proxy в Raspberry Pi, перенесите файлы в домашнюю папку пользователя (например, `/home/pi/` или `/home/ney/`) с помощью программ: **Bitvise**, **FileZilla** или **ForkLift** (для macOS). Прошивка сама отсканирует эту папку и предложит их импортировать в нужных утилитах. Убедитесь, что в именах файлов **нет пробелов и круглых скобок!**

## 4. 🔑 Менеджеры Конфигов (VPN, Proxy, Cloak, SSH)

Ниже перечислены основные команды, которые позволят вам загружать и запускать защищенные туннели. Все они имеют одинаковый синтаксис FZF-меню (введите команду, чтобы увидеть список ваших файлов).

### OpenVPN
- **Команда:** `ovpnney` 
- **Формат конфига:** `.ovpn`
- **Описание:** Менеджер OpenVPN. Поддерживает TCP/UDP. Загрузите файлы `.ovpn` в `/home/pi/`, запустите `ovpnney`, нажмите `ctrl-i` (импорт) и выберите конфиг стрелочками. 

### WireGuard & AmneziaWG
- **Команда:** `wgney`
- **Формат конфига:** `.conf`
- **Описание:** Мощный клиент для WireGuard и AmneziaWG (с обфускацией `Jc`, `Jmin`, `S1`, `H1` и т.д.). Импортируйте `.conf` файлы аналогичным образом. Отрабатывает очень быстро и не нагружает процессор.

### Cloak Client (Анти-DPI)
- **Команда:** `ckney`
- **Формат конфига:** `.json`
- **Описание:** Менеджер подключения к серверам через Cloak (обычно используется поверх Shadowsocks) для полной маскировки трафика. Поддерживает режим автозагрузки с UDP или без UDP.
  - Положите `config.json` в домашнюю директорию.
  - Наберите `ckney manager`, выберите конфиг.

### Shadowsocks
- **Команда:** `ssney`
- **Формат конфига:** `.json`
- **Описание:** Запускает Shadowsocks-клиент (в связке с `proxychains` или `iptables` редиректами). Управление профилями аналогично Cloak.

### Доп. Протоколы: L2TP, PPTP, IKEv2, SSTP
- `l2tpney` — Менеджер подключений L2TP (без IPsec и с IPsec `ipsec.secrets` / `ipsec.conf`).
- `pptpney` — Менеджер архаичного, но иногда нужного протокола PPTP.
- `ikev2ney` — Управление IKEv2/IPSec туннелями.
- `sstpney` — Управление подключением SSTP.

### SSH-Туннели 
- `sshney` — Создает защищенный SOCKS-туннель через удаленный SSH сервер (Dynamic Port Forwarding). Аналог `ssh -D`. Помогает обойти блокировки за счет простого хостинга. Управляет файлами `.json`, в которых хранятся данные от серверов (IP, порт, логин).

### Proxy-Туннели
- `proxyney` / `proxydney` — Службы для роутинга трафика в SOCKS4/SOCKS5 прокси. `proxydney` позволяет настроить каскадные цепочки. Интегрировано с `redsocks`/`iptables`.

## 5. Системные и Вспомогательные Утилиты

### IPNey — Мощный сетевой инструмент и Kill-Switch
- **Команда:** `ipney`
- Основные ключи:
  - `ipney killon` — Включает **Kill-Switch**. И уже работает с момента запуска через **`ney-killswitch.service`**! Трафик пойдет **только** через запущенные `tun0` (VPN).
  - `ipney killoff` — Деактивирует Kill-Switch до следующей перезагрузки, возвращая трафик в LAN.
  - `ipney lip <айпи>` — Меняет ваш локальный IPv4 адрес подсети для мостов `Br0`.
  - `ipney hostname <имя>` — Меняет системное имя (Hostname) Raspberry Pi.
  - `ipney ipv6-manager` — Управление утечками IPv6. Полное отключение IPv6 на выбранных сетевых интерфейсах.

### Сеть и MAC адреса
- `wifiney` — Менеджер защищенной Wi-Fi точки (Hostpot). Позволяет сменить Имя сети (SSID), пароль, канал, а также сделать сеть **скрытой / видимой**. Автоматически раздает DHCP-адреса через интерфейс `wlan1`.
- `macney` — Позволяет рандомизировать или жестко сменить MAC (физический) адрес ваших Wi-Fi свистков или LAN портов для защиты от трекинга (полезно в общественных сетях).
- `dnsney` — Модуль для защиты от DNS-утечек. Меняет DNS-резолвер на мостах на Cloudflare (`1.1.1.1`), Google (`8.8.8.8`) или кастомные.

### Служба Сброса
- `resetney` (или `reset`) — Скрипт "Глубокой очистки". Если вы запутались в настройках интерфейсов, мостов `br0`/`br1` или хотите обновить параметры Wi-Fi по умолчанию — запустите эту команду и подтвердите действие. Она пересоберет маршруты, восстановит интернет и попросит перезагрузить систему.

---
> [!TIP]
> **Типичный сценарий использования:**
> 1. Включить Raspberry Pi.
> 2. Дождаться загрузки. (Благодаря `ney-killswitch.service` Kill-Switch уже включен, трафик наружу не идет).
> 3. Зайти по SSH, загрузить свой файл `config.ovpn` через SFTP.
> 4. Написать в терминале: `ovpnney`, импортировать `config.ovpn` и нажать Enter.
> 5. Проверить статус через: `infoney`. Радоваться 100% анонимности!
